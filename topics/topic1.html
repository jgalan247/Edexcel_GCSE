1A: Computational Thinking and Algorithms

Key Definition

Computational Thinking is a set of problem-solving methods that involve expressing problems and their solutions in ways that a computer could execute.

Computational Thinking Principles

There are four key principles of computational thinking that you need to understand:

Decomposition

Breaking down a complex problem into smaller, more manageable sub-problems. Each sub-problem can then be solved individually.

Abstraction

Focusing on the important information only, ignoring irrelevant detail. This helps simplify the problem.

Pattern Recognition

Looking for similarities among and within problems. Patterns can be reused to solve similar problems.

Algorithm Design

Developing a step-by-step solution to the problem, or the rules to follow to solve the problem.

What is an Algorithm?

Definition

An algorithm is a sequence of steps that can be followed to complete a task. Algorithms must be unambiguous and have a defined start and end point.

Algorithms can be represented in several ways:

  • Pseudocode - A structured way of writing algorithms using plain English-like statements
  • Flowcharts - Visual diagrams showing the flow of a program
  • Program code - The actual implementation in a programming language

Flowchart Symbols

Flowcharts use standard symbols to represent different operations:

Symbol Name Purpose
Oval/Rounded Rectangle Terminator Start or End of algorithm
Rectangle Process Instruction or calculation
Diamond Decision Yes/No question (selection)
Parallelogram Input/Output Data input or output
Arrow Flow Line Direction of flow

Searching Algorithms

Linear Search

Linear search (or sequential search) checks each item in the list one by one until the target is found or the end is reached.

Key Points

  • Works on both sorted and unsorted lists
  • Time complexity: O(n) - worst case checks every item
  • Simple to implement but inefficient for large datasets
# Linear Search in Python
def linear_search(list, target):
    for i in range(len(list)):
        if list[i] == target:
            return i  # Return position if found
    return -1  # Return -1 if not found

# Example usage
numbers = [4, 2, 7, 1, 9, 3]
result = linear_search(numbers, 7)
print(f"Found at index: {result}")  # Output: 2

Binary Search

Binary search works by repeatedly dividing the search space in half. It requires a sorted list.

Important

Binary search only works on sorted data. If the data is not sorted, you must sort it first or use linear search instead.

# Binary Search in Python
def binary_search(list, target):
    low = 0
    high = len(list) - 1

    while low <= high:
        mid = (low + high) // 2

        if list[mid] == target:
            return mid  # Found it!
        elif list[mid] < target:
            low = mid + 1  # Search right half
        else:
            high = mid - 1  # Search left half

    return -1  # Not found

# Example usage
sorted_numbers = [1, 2, 3, 4, 7, 9]
result = binary_search(sorted_numbers, 7)
print(f"Found at index: {result}")  # Output: 4

Sorting Algorithms

Bubble Sort

Bubble sort repeatedly steps through the list, compares adjacent elements and swaps them if they are in the wrong order. The largest values "bubble" to the end of the list.

# Bubble Sort in Python
def bubble_sort(list):
    n = len(list)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1 - i):
            if list[j] > list[j + 1]:
                # Swap elements
                list[j], list[j + 1] = list[j + 1], list[j]
                swapped = True
        if not swapped:
            break  # List is sorted
    return list

# Example
numbers = [64, 34, 25, 12, 22, 11, 90]
sorted_numbers = bubble_sort(numbers)
print(sorted_numbers)  # [11, 12, 22, 25, 34, 64, 90]

Merge Sort

Merge sort uses a divide and conquer approach. It divides the list into halves, recursively sorts them, then merges the sorted halves.

Comparison

Feature Bubble Sort Merge Sort
Time Complexity O(n²) O(n log n)
Space In-place Needs extra memory
Best for Small lists Large datasets

Constructing Truth Tables

For complex Boolean expressions with multiple gates, you need to work through each gate step by step.

Exam Tip

When constructing truth tables:

  1. List all possible combinations of inputs (2ⁿ rows for n inputs)
  2. Work through intermediate gates from left to right
  3. Add columns for intermediate outputs if helpful
  4. Calculate the final output last

Example: Q = (A AND B) OR (NOT C)
