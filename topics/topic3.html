3A: Hardware Components

Key Definition

Hardware refers to the physical components of a computer system that you can see and touch, such as the CPU, memory, and storage devices.

The CPU (Central Processing Unit)

Key Definition

The CPU is the brain of the computer. It processes instructions from programs by performing the fetch-decode-execute cycle billions of times per second.

CPU Components

Control Unit (CU)

Controls and coordinates the activities of the CPU. Fetches instructions from memory, decodes them, and sends control signals to other components.

Arithmetic Logic Unit (ALU)

Performs all arithmetic calculations (add, subtract, multiply, divide) and logical operations (AND, OR, NOT comparisons).

Cache Memory

Very fast, small memory inside the CPU. Stores frequently used instructions and data for quick access. Faster than RAM but more expensive.

Registers

Tiny, extremely fast storage locations within the CPU. Hold data currently being processed. Examples: Accumulator, Program Counter, MAR, MDR.

The Fetch-Decode-Execute Cycle

  1. Fetch: The next instruction is fetched from RAM using the address in the Program Counter (PC)
  2. Decode: The Control Unit decodes the instruction to determine what needs to be done
  3. Execute: The instruction is carried out (may involve the ALU for calculations)

CPU Performance Factors

Factor Description Effect
Clock Speed Measured in GHz. Number of FDE cycles per second. Higher = faster processing
Number of Cores Each core can process instructions independently. More cores = better multitasking
Cache Size Amount of high-speed memory in the CPU. Larger = fewer delays fetching data

Memory

RAM (Random Access Memory)

RAM Characteristics

  • Volatile - Contents are lost when power is turned off
  • Read and write - Data can be changed
  • Fast access - Quicker than secondary storage
  • Stores: Currently running programs and data being used

ROM (Read Only Memory)

ROM Characteristics

  • Non-volatile - Contents are retained when power is off
  • Read only - Data cannot be changed (in basic ROM)
  • Stores: BIOS/boot instructions needed to start the computer

Virtual Memory

When RAM is full, the operating system uses part of the hard drive as temporary RAM. This is called virtual memory.

Drawback

Virtual memory is much slower than RAM because hard drives are slower than memory chips. Too much reliance on virtual memory causes the computer to slow down significantly.

Secondary Storage

Key Definition

Secondary storage is non-volatile storage used to store data and programs permanently. It retains data even when the computer is switched off.

Types of Secondary Storage

Magnetic Storage

Examples: Hard Disk Drives (HDD), magnetic tape

How it works: Data stored as magnetic patterns on spinning platters

Pros: High capacity, low cost per GB

Cons: Slower, moving parts can fail, fragile

Solid State Storage

Examples: SSD, USB flash drives, SD cards

How it works: Data stored in flash memory chips (no moving parts)

Pros: Very fast, durable, silent, low power

Cons: More expensive per GB, limited write cycles

Optical Storage

Examples: CD, DVD, Blu-ray

How it works: Laser reads/writes patterns of pits and lands on disc surface

Pros: Cheap, portable, good for distributing media

Cons: Limited capacity, slow, easily scratched

Cloud Storage

Data stored on remote servers accessed via the internet.

Advantages Disadvantages
Access from anywhere with internet Requires internet connection
Automatic backups Ongoing subscription costs
Easy to share files Security/privacy concerns
Scalable storage Reliance on third party

Embedded Systems

Key Definition

An embedded system is a computer system built into a larger device to perform a specific dedicated function. It typically has limited resources and runs specialized software.

Examples: Washing machines, traffic lights, smartwatches, car engine management systems, digital cameras, smart thermostats.

3B: Software

Key Definition

Software is a collection of programs and data that tell the computer how to perform specific tasks. Software is intangible - you cannot physically touch it.

Types of Software

System Software

Software that manages and controls the computer hardware, providing a platform for application software to run.

Examples: Operating systems, utility software, device drivers

Application Software

Software designed to help users perform specific tasks or solve particular problems.

Examples: Word processors, web browsers, games, spreadsheets

Operating Systems

Key Definition

An operating system (OS) is system software that manages computer hardware and software resources and provides common services for computer programs. It acts as an interface between the user and the hardware.

Functions of an Operating System

Memory Management

Allocates RAM to programs, manages virtual memory, and ensures programs don't interfere with each other's memory space.

Process Management

Manages running programs (processes), allocates CPU time, handles multitasking between applications.

File Management

Organises files and folders on storage devices, controls access permissions, handles reading and writing data.

Peripheral Management

Controls input/output devices through device drivers, manages print queues, handles data transfer.

User Management

Handles user accounts, login authentication, access control, and security settings.

User Interface

Provides GUI (graphical) or CLI (command line) interface for users to interact with the computer.

Utility Software

Key Definition

Utility software performs maintenance and housekeeping tasks to help the computer run efficiently. It supports, enhances, or protects the computer system.

Utility Purpose
Antivirus/Anti-malware Scans for, detects, quarantines, and removes malicious software
Disk Defragmentation Reorganises fragmented files on HDD to improve access speed (not needed for SSDs)
Backup Software Creates copies of data to protect against data loss
Encryption Software Scrambles data so it can only be read with the correct key
Compression Software Reduces file sizes for storage or transmission
Firewall Monitors and controls incoming/outgoing network traffic

Programming Languages

Levels of Programming Language

Low-Level Languages

Machine Code: Binary instructions directly executed by CPU. Specific to processor architecture.

Assembly Language: Uses mnemonics (short codes) instead of binary. Needs an assembler to convert to machine code.

Pros: Fast execution, direct hardware control, efficient use of memory

Cons: Difficult to learn/write, processor-specific, time-consuming

High-Level Languages

Examples: Python, Java, C++, JavaScript

Uses English-like statements. Needs a compiler or interpreter to convert to machine code.

Pros: Easier to learn/write, portable across platforms, faster development

Cons: Slower execution, less direct hardware control, requires translator

Translators

Translator Function Characteristics
Compiler Translates entire high-level program into machine code before execution Creates executable file, faster execution, errors reported at end
Interpreter Translates and executes code line by line No executable file, slower execution, stops at first error
Assembler Translates assembly language into machine code One-to-one translation, processor specific

June 2023 - Paper 1 - Q9 4 marks

Describe two functions of an operating system. [4 marks]

Show Mark Scheme Answer

Any two from (2 marks each - 1 for naming, 1 for description):

  • Memory management: Allocates RAM to programs and manages virtual memory when RAM is full
  • Process/processor management: Allocates CPU time to running programs and manages multitasking
  • File management: Organises files into folders, controls access permissions, handles saving/loading
  • User management: Handles user accounts, passwords, and access rights
  • Peripheral management: Controls input/output devices through device drivers
  • User interface: Provides GUI or CLI for users to interact with the computer
June 2022 - Paper 1 - Q10 4 marks

Compare the use of a compiler and an interpreter. [4 marks]

Show Mark Scheme Answer

Compiler:

  • Translates the entire program before execution (1 mark)
  • Creates an executable file that can be run without the compiler (1 mark)
  • Faster execution once compiled (1 mark)
  • Reports all errors at end of compilation (1 mark)

Interpreter:

  • Translates and executes code one line at a time (1 mark)
  • No executable file created - needs interpreter each time (1 mark)
  • Slower execution (1 mark)
  • Stops at first error encountered (1 mark)
