4A: Networks, Topologies and Protocols

Key Definition

A computer network is two or more computers connected together to share resources, data, and communicate with each other.

Types of Networks

LAN (Local Area Network)

A network covering a small geographical area, such as a single building or school.

  • Usually owned by a single organisation
  • Uses Ethernet cables or Wi-Fi
  • Fast data transfer speeds
  • Hardware maintained in-house

WAN (Wide Area Network)

A network covering a large geographical area, often across cities or countries.

  • Connects multiple LANs together
  • Uses telephone lines, fibre optic, satellites
  • The internet is the largest WAN
  • Often uses third-party infrastructure

Client-Server vs Peer-to-Peer

Feature Client-Server Peer-to-Peer (P2P)
Structure Central server provides resources to clients All computers are equal; share resources directly
Security Centrally managed, more secure Each user manages own security, less secure
Backup Centralised backup on server Each user responsible for own backup
Cost Expensive server hardware and admin Cheaper, no dedicated server needed
Failure Server failure affects entire network One computer failing doesn't affect others

Network Topologies

Key Definition

Network topology refers to the arrangement and layout of computers and connections in a network. It can be physical (actual layout) or logical (how data flows).

Star Topology

All devices connect to a central switch or hub.

Advantages:

  • One cable failure only affects one device
  • Easy to add new devices
  • Fast performance, no collisions

Disadvantages:

  • Central switch failure = entire network down
  • More cabling required than bus
  • Expensive central hardware

Mesh Topology

Every device connects to every other device (full mesh) or some devices connect to multiple others (partial mesh).

Advantages:

  • Very reliable - multiple paths available
  • No single point of failure
  • Fast data transmission

Disadvantages:

  • Very expensive (lots of cabling)
  • Complex to set up and manage
  • Difficult to troubleshoot

Network Hardware

Device Purpose
Network Interface Card (NIC) Hardware that allows a computer to connect to a network. Has a unique MAC address.
Switch Connects devices in a network and sends data only to the intended recipient using MAC addresses.
Router Connects different networks together and routes data between them using IP addresses. Connects LAN to the internet.
Wireless Access Point (WAP) Allows wireless devices to connect to a wired network using Wi-Fi.
Transmission Media Ethernet cables (copper), fibre optic cables (light), or wireless (radio waves).

The Internet and the World Wide Web

Common Misconception

The Internet and the World Wide Web are NOT the same thing!

  • The Internet is the global network of connected computers and infrastructure
  • The World Wide Web is a collection of websites and web pages accessed via the internet using HTTP

DNS (Domain Name System)

The DNS translates human-readable domain names (like www.google.com) into IP addresses (like 142.250.187.46) that computers use to identify each other.

  1. User types URL in browser
  2. Browser asks DNS server for the IP address
  3. DNS server returns the IP address
  4. Browser connects to the web server using the IP address
  5. Web server sends the requested web page

Network Protocols

Key Definition

A protocol is a set of rules that govern how devices communicate over a network. Protocols ensure all devices can understand each other.

Protocol Purpose
TCP/IP Transmission Control Protocol/Internet Protocol - the foundation of internet communication. TCP breaks data into packets; IP handles addressing and routing.
HTTP/HTTPS Hypertext Transfer Protocol - used for transferring web pages. HTTPS adds encryption for security.
FTP File Transfer Protocol - used to transfer files between computers over a network.
SMTP Simple Mail Transfer Protocol - used to send emails.
POP3/IMAP Used to receive and manage emails. IMAP syncs across devices; POP3 downloads to one device.

IP Addresses and MAC Addresses

IP Address

A logical address assigned to a device on a network.

  • IPv4: 32-bit, e.g., 192.168.1.1
  • IPv6: 128-bit, e.g., 2001:0db8:85a3::8a2e
  • Can change (dynamic) or be fixed (static)
  • Used for routing between networks

MAC Address

A physical address burned into the NIC.

  • 48-bit, e.g., 00:1A:2B:3C:4D:5E
  • Unique to each device
  • Cannot be changed (permanent)
  • Used for communication within a LAN

4B: Network Security

Key Definition

Cyber security is the practice of protecting computer systems, networks, and data from digital attacks, unauthorised access, and damage.

Types of Malware

Key Definition

Malware (malicious software) is any software designed to damage, disrupt, or gain unauthorised access to a computer system.

Virus

Malicious code that attaches itself to legitimate programs. Spreads when the infected program is run. Can delete files, corrupt data, or slow down systems.

Worm

Self-replicating malware that spreads across networks without needing a host program. Can consume bandwidth and crash systems.

Trojan

Malware disguised as legitimate software. Users are tricked into installing it. Can create backdoors for hackers.

Ransomware

Encrypts user's files and demands payment for the decryption key. Often spreads via email attachments.

Spyware

Secretly monitors user activity and collects personal information. Can capture passwords and banking details.

Adware

Displays unwanted advertisements. Often bundled with free software. Can slow down systems and track browsing habits.

Social Engineering Attacks

Key Definition

Social engineering is the manipulation of people into giving away confidential information or performing actions that compromise security. It exploits human psychology rather than technical vulnerabilities.

Attack Type Description
Phishing Fake emails/websites that appear legitimate, designed to trick users into revealing passwords or personal information.
Shoulder Surfing Watching someone enter their password or PIN by looking over their shoulder.
Blagging (Pretexting) Creating a false scenario to manipulate someone into providing information or access.
Pharming Redirecting users from legitimate websites to fraudulent ones by corrupting DNS.

Other Network Threats

Brute Force Attack

Systematically trying every possible password combination until the correct one is found. Stronger passwords take longer to crack.

Denial of Service (DoS/DDoS)

Overwhelming a server with traffic so legitimate users cannot access it. DDoS uses multiple computers (often a botnet).

SQL Injection

Inserting malicious SQL code into input fields to manipulate databases, potentially accessing or deleting data.

Man-in-the-Middle

Intercepting communications between two parties without their knowledge, potentially altering or stealing data.

Security Measures

Prevention Methods

Measure How It Works
Firewall Monitors and filters incoming/outgoing network traffic based on security rules. Blocks unauthorised access.
Antivirus/Anti-malware Scans for known malware signatures, monitors for suspicious behaviour, quarantines and removes threats.
Encryption Scrambles data so only authorised users with the decryption key can read it. Protects data in transit and at rest.
Strong Passwords Long, complex passwords with uppercase, lowercase, numbers, and symbols. Harder to crack with brute force.
Two-Factor Authentication (2FA) Requires two forms of verification (e.g., password + phone code). Even if password is stolen, account remains protected.
Access Control Limits what users can access based on their role. Users only have permissions they need.
Regular Updates Software patches fix known vulnerabilities. Keeping systems updated prevents exploitation of known bugs.
Backups Regular copies of data stored securely. Allows recovery from ransomware or data loss. 
# Python: Simple password strength checker
def check_password_strength(password):
    score = 0
    feedback = []

    if len(password) >= 8:
        score += 1
    else:
        feedback.append("Use at least 8 characters")

    if any(c.isupper() for c in password):
        score += 1
    else:
        feedback.append("Add uppercase letters")

    if any(c.islower() for c in password):
        score += 1
    else:
        feedback.append("Add lowercase letters")

    if any(c.isdigit() for c in password):
        score += 1
    else:
        feedback.append("Add numbers")

    if any(c in "!@#$%^&*()_+-=" for c in password):
        score += 1
    else:
        feedback.append("Add special characters")

    return score, feedback

# Example usage
password = "MyP@ssw0rd!"
score, tips = check_password_strength(password)
print(f"Strength: {score}/5")

How to protect against network threats
June 2023 - Paper 1 - Q14 6 marks

(a) Define the term 'phishing'. [2 marks]

(b) Describe two ways an organisation can protect itself from cyber attacks. [4 marks]

Show Mark Scheme Answer

(a) Phishing is:

  • A type of social engineering attack (1 mark)
  • Where fake emails/websites trick users into revealing sensitive information such as passwords or bank details (1 mark)

(b) Any two methods with description (2 marks each):

  • Firewall: Monitors network traffic and blocks unauthorised access based on security rules
  • Antivirus: Scans for malware, detects suspicious behaviour, and removes threats
  • Encryption: Scrambles data so it cannot be read without the decryption key
  • Staff training: Educates employees to recognise phishing and other attacks
  • Regular updates: Patches software vulnerabilities that hackers could exploit
  • Strong password policies: Requires complex passwords and regular changes
June 2022 - Paper 1 - Q15 4 marks

Explain what a denial of service (DoS) attack is and describe how it affects a business. [4 marks]

Show Mark Scheme Answer

What it is:

  • An attack that floods a server with requests/traffic (1 mark)
  • Making it unable to respond to legitimate users (1 mark)

Effects on business:

  • Website/services become unavailable to customers (1 mark)
  • Loss of sales/revenue during the attack (1 mark)
  • Damage to reputation (1 mark)
  • Cost of recovering and improving security (1 mark)
